# AI 企业级项目实战

为了规范 [Vibe Coding](https://mp.weixin.qq.com/s/eVJtcqXrivivfOwWN_Y0_w)，过去一段时间「Spec-Driven」的开发理念很火， 也出现了 speckit、openspec 等开源框架，网上也都是一些从零开发、演示一些「玩具」项目，对于公司级的旧项目怎么引入很少分享。

腾讯的这篇文章 [认知重建：Speckit 用了三个月，我放弃了——走出工具很强但用不好的困境](https://mp.weixin.qq.com/s/CXx-0ar1EBf14vgQHHjU7A) ，深度分享了从踩坑规范驱动工具，到借鉴 Anthropic 多 Agent 协作架构、融合上下文工程与复合工程理念，最终实现边际成本递减、知识持续复利的完整历程。

这是目前看到的 AI 企业级实践最好的文章，看到了未来 AI 编程的模样。

# 最顶格的 AI 对话

1 月 10 日在中国举办的 AGI-Next 前沿峰会汇聚了中国大模型领域的顶尖专家和学者，包括唐杰、杨植麟、林俊旸、姚顺雨等，探讨了大模型的发展方向、技术挑战和中国 AI 的未来。

唐杰：清华大学教授、智谱创始人。

杨植麟：月之暗面 Kimi 创始人、CEO。

林俊旸：Qwen 技术负责人。

姚顺雨：腾讯首席科学家 （98 年）。

张钹：清华大学教授、院士（91 岁高龄）。

通过 [ 2026开年最顶格的AI对话：当唐杰、杨植麟、林俊旸、姚顺雨聚在一起，他们看见了什么？](https://mp.weixin.qq.com/s/L8d3LadTuIBSZoxynwoAXg) 可以了解到这些最前沿的人对 AI 的看法和现状。

# Anthropic 官方博客

最近被 Anthropic 新出的 Coworker 刷屏，他们公司不仅有先进的模型，各种 Agent 开发方法论也是特别值得学习的，作者总结了Anthropic 博客 Agent 文章的阅读顺序，做了导读。

[https://zhuanlan.zhihu.com/p/1994463393139151039](https://zhuanlan.zhihu.com/p/1994463393139151039)

公众号也看到有人翻译的一篇 [Anthropic 万字长文：AI Agent 评估体系全解析](https://mp.weixin.qq.com/s/pwhg5Kmh3wMlxqIlbVqGMw) ，过去写程序都有明确的输入输出，但现在 Agent  输入和输出都变得不确定，如何进行评估也成了一个需要探讨的问题。

# Skills 推荐

[Skills](https://mp.weixin.qq.com/s/JsWUMirBwwmILp8xZDwNgQ) 持续爆火中，Vercel 官方出了 React 最佳实践 Skills ，列了 8 大类的优化方向。 虽然是给 AI 用的，但如果平常自己也写 React 去看一看也不错。

[https://github.com/vercel-labs/agent-skills](https://github.com/vercel-labs/agent-skills)

还有 obsidian-skills，由 Obsidian 的 CEO Steph Ango 亲自开源的项目，专门用于教 Claude Code 如何正确地理解、编写和管理 Obsidian 的笔记文件。

[https://github.com/kepano/obsidian-skills](https://github.com/kepano/obsidian-skills)

专注下上下文管理的的 Skills，Agent-Skills-for-Context-Engineering 

[https://github.com/muratcankoylan/Agent-Skills-for-Context-Engineering](https://github.com/muratcankoylan/Agent-Skills-for-Context-Engineering) 

号称复现 Manus 「价值 $2 billion” 的工作流」  planning-with-files。

[https://github.com/OthmanAdi/planning-with-files](https://github.com/OthmanAdi/planning-with-files)

这些 Skill 是最近出现在我视野中的，但因为我也没用过，效果和安全性不敢保证，但未来编写自己相关的 Skills 时候一定是不错的参考资料。
